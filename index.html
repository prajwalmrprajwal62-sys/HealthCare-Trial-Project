How It Works

A 3-layer edge AI pipeline. Role-based outputs. All on-device.

INPUT

  • 📷 Camera — Fundus images
  • 🎙 Voice — Kannada (Bhashini)
  • 📊 Manual Vitals — BP, Glucose
  • 💓 Camera PPG — Heart rate
→

AI ENGINE

  • 🧠 YOLOv8-n: Fundus (<20ms)
  • 👁 RETFound: DR grading
  • 📊 XGBoost+SHAP (AUC 0.977)
  • 🗣 Bhashini: 22 languages
  • 🤖 Sarvam AI: Summaries
  • 💓 Camera PPG: Signal proc.
→

OUTPUT

  • 🔴 Risk Score → ASHA Worker
  • 📄 Referral Report → Hospital
  • 📋 Research Report → Doctor
  • 📝 Summary → Patient
  • 💾 ABHA sync → Health Locker
  • 🥗 Diet plan → Patient
  • 📊 Dashboard → PHC Supervisor
✓ 100% Offline Core✓ Syncs to ABHA when online✓ Works on ₹3,000 Android

Architecture Layers

Layer 1 — Frontend (React)

Patient Interface | Hospital Dashboard | ASHA Worker Interface

Layer 2 — Backend (Node.js + Express)

Auth, role-based APIs, hospital search, appointments, notifications

Layer 3 — Database (MongoDB)

Users, hospitals, beds, appointments, reports, reminders, schemes

Layer 4 — ML Services (Python)

YOLO report analysis, OpenCV crowd detection, FastAPI wrapper APIs
